üîÑ Complete Data Flow

1
Heartbeat Registration
Student kiosk sends POST to /api/system-heartbeat every 30 seconds
2
Database Update
Server upserts SystemRegistry with systemNumber, labId, socketId, lastSeen
3
Admin Views Systems
Admin opens modal, fetches GET /api/lab-systems/CC1, sees all 69 systems
4
Admin Selects Systems
Admin checks boxes for systems 1, 5, 10, 15, 23 (or clicks "Select All")
5
Shutdown Command Sent
Admin clicks "Shutdown", POST /api/shutdown-systems with systemNumbers array
6
Server Processes Request
Server queries SystemRegistry, emits 'force-shutdown-system' to each socketId
7
Kiosk Receives Command
Student kiosk listens to Socket.IO event, shows full-screen red warning
8
Shutdown Execution
After 5s countdown, executes: shutdown /s /f /t 0 (immediate Windows shutdown)
